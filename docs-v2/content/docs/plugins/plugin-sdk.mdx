The `@opentabs-dev/plugin-sdk` package provides everything needed to build an OpenTabs plugin: the `OpenTabsPlugin` base class, the `defineTool()` factory, and the `ToolError` class for structured error reporting.

```bash
bun add @opentabs-dev/plugin-sdk
```

## OpenTabsPlugin Base Class

Every plugin extends `OpenTabsPlugin` and exports a single instance as the default export:

```typescript
import { OpenTabsPlugin } from '@opentabs-dev/plugin-sdk';
import type { ToolDefinition } from '@opentabs-dev/plugin-sdk';

class MyPlugin extends OpenTabsPlugin {
  readonly name = 'my-plugin';
  readonly version = '0.0.1';
  readonly description = 'Give AI agents access to My App';
  readonly displayName = 'My App';
  readonly urlPatterns = ['*://*.myapp.com/*'];
  readonly tools: ToolDefinition[] = [/* ... */];

  async isReady(): Promise<boolean> {
    // Check if the user is authenticated
    return document.cookie.includes('session=');
  }
}

export default new MyPlugin();
```

### Abstract Properties

| Property | Type | Description |
| --- | --- | --- |
| `name` | `string` | Unique plugin identifier. Lowercase alphanumeric with hyphens (`^[a-z0-9]+(-[a-z0-9]+)*$`). Used as the tool name prefix (e.g., `slack` → `slack_send_message`). Reserved names: `system`, `browser`, `opentabs`, `extension`, `config`, `plugin`, `tool`, `mcp`. |
| `version` | `string` | Semver version string (e.g., `0.0.1`). |
| `description` | `string` | Brief description of the plugin's purpose. Shown to AI agents in MCP tool listings. |
| `displayName` | `string` | Human-readable name shown in the Chrome extension side panel and health endpoint (e.g., `Slack`). |
| `urlPatterns` | `string[]` | Chrome match patterns that determine which tabs get the adapter injected. See [Chrome match patterns](https://developer.chrome.com/docs/extensions/develop/concepts/match-patterns). |
| `tools` | `ToolDefinition[]` | Array of tool definitions created with `defineTool()`. At least one tool is required. |

### Abstract Methods

#### `isReady(): Promise<boolean>`

Readiness probe called by the Chrome extension to determine if the web app in the current tab is ready to accept tool requests. Runs in the page's MAIN world context with full access to DOM, cookies, and JavaScript globals.

Returns `true` if the user is authenticated and the service is operational. The result maps to the [tab state machine](/docs/concepts/tab-state-machine):

| Condition | Tab State |
| --- | --- |
| No tab matches `urlPatterns` | `closed` |
| Tab exists, `isReady()` returns `false` | `unavailable` |
| Tab exists, `isReady()` returns `true` | `ready` |

The extension probes `isReady()` with a 5-second timeout. If the probe times out or throws, the tab is treated as `unavailable`.

**Example** — checking for a session cookie:

```typescript
async isReady(): Promise<boolean> {
  return document.cookie.includes('session_token=');
}
```

**Example** — checking for a DOM element that only appears after authentication:

```typescript
async isReady(): Promise<boolean> {
  return document.querySelector('[data-user-id]') !== null;
}
```

**Example** — waiting for SPA hydration with retries:

```typescript
async isReady(): Promise<boolean> {
  if (isAuthenticated()) return true;
  // Retry for up to 3 seconds for SPA hydration
  return waitForAuth({ timeout: 3000, interval: 500 });
}
```

### Optional Methods

#### `teardown(): void`

Called by the platform before re-injection (during a `plugin.update`) to allow cleanup of event listeners, timers, or global state set up by the previous adapter version. Plugins that do not set up persistent side effects can omit this method.

```typescript
class MyPlugin extends OpenTabsPlugin {
  private observer: MutationObserver | null = null;

  // ... other properties ...

  teardown(): void {
    this.observer?.disconnect();
    this.observer = null;
  }
}
```

## defineTool()

The `defineTool()` factory creates type-safe tool definitions with full generic inference:

```typescript
import { defineTool } from '@opentabs-dev/plugin-sdk';
import { z } from 'zod';

export const sendMessage = defineTool({
  name: 'send_message',
  displayName: 'Send Message',
  description: 'Send a message to a channel or thread',
  icon: 'send',
  input: z.object({
    channel: z.string().min(1).describe('Channel ID (e.g., C01234567)'),
    text: z.string().min(1).describe('Message text to send'),
    thread_ts: z.string().optional().describe('Thread timestamp to reply in a thread'),
  }),
  output: z.object({
    channel: z.string().describe('Channel ID the message was posted to'),
    ts: z.string().describe('Timestamp of the posted message'),
  }),
  handle: async (params) => {
    const response = await fetch('/api/messages', {
      method: 'POST',
      body: JSON.stringify(params),
    });
    const data = await response.json();
    return { channel: data.channel, ts: data.ts };
  },
});
```

### Configuration Properties

| Property | Type | Description |
| --- | --- | --- |
| `name` | `string` | Tool name in `snake_case`. Auto-prefixed with the plugin name when registered as an MCP tool (e.g., `send_message` → `slack_send_message`). |
| `displayName` | `string` | Human-readable name shown in the side panel (e.g., `Send Message`). |
| `description` | `string` | Description shown to AI agents. Must be accurate and informative — AI agents rely on this to choose the right tool. Maximum 1000 characters. |
| `icon` | `LucideIconName` | Kebab-case [Lucide](https://lucide.dev/icons) icon name displayed in the side panel (e.g., `send`, `message-square`, `search`). |
| `input` | `z.ZodObject` | Zod object schema defining the tool's input parameters. Serialized to JSON Schema for MCP registration and manifest generation. |
| `output` | `z.ZodType` | Zod schema defining the tool's return value shape. Serialized to JSON Schema for manifest generation. |
| `handle` | `(params: TInput) => Promise<TOutput>` | Async function that executes the tool. Receives pre-validated input. Runs in the browser page's MAIN world context. |

### handle() Execution Context

The `handle()` function runs in the browser page's MAIN world via `chrome.scripting.executeScript`. This means it has full access to:

- **DOM**: `document.querySelector()`, `document.createElement()`, etc.
- **Cookies**: `document.cookie` (same-origin cookies for the page)
- **localStorage / sessionStorage**: the page's storage
- **fetch**: makes requests with the page's session cookies (same-origin)
- **window globals**: any JavaScript variables the page exposes

The function does **not** have access to Chrome extension APIs (`chrome.*`), Node.js APIs, or Bun APIs. It runs in a standard browser environment.

The execution timeout is 25 seconds (5 seconds less than the server-side 30-second dispatch timeout, ensuring a response always reaches the server).

## Zod Schema Requirements

Tool input and output schemas are serialized to JSON Schema via `z.toJSONSchema()` for the MCP protocol and plugin manifest. Keep schemas compatible with JSON Schema serialization:

### Do

- Use primitives: `z.string()`, `z.number()`, `z.boolean()`
- Use objects: `z.object({ ... })`
- Use arrays: `z.array(z.string())`
- Use optionals: `z.string().optional()`
- Use unions: `z.union([z.string(), z.number()])`
- Use literals: `z.literal('active')`
- Use enums: `z.enum(['asc', 'desc'])`
- Use `.describe()` for parameter descriptions
- Use `.min()`, `.max()`, `.regex()` for validation

### Don't

- **Never use `.transform()`** — Zod transforms cannot be represented in JSON Schema. If input needs normalization, do it in the `handle()` function.
- **Never use `.pipe()` or `.preprocess()`** — these produce runtime-only behavior that `z.toJSONSchema()` cannot serialize.
- **`.refine()` callbacks must never throw** — Zod 4 runs `.refine()` callbacks even when the preceding validator has already failed (e.g., `z.url().refine(fn)` calls `fn` even on non-URL strings). Always wrap potentially-throwing code in try-catch and return `false` on error:

```typescript
// BAD — throws if input is not a valid URL
z.string().url().refine((s) => new URL(s).hostname.endsWith('.slack.com'))

// GOOD — catches errors from invalid input
z.string().url().refine((s) => {
  try { return new URL(s).hostname.endsWith('.slack.com'); }
  catch { return false; }
})
```

If the schema causes `z.toJSONSchema()` to throw, the `opentabs build` CLI will fail. The fix is always to simplify the schema and move logic to the `handle()` function.

## ToolError

The `ToolError` class provides structured error reporting from tool handlers. The platform catches these and returns them as MCP error responses with a machine-readable `code` field:

```typescript
import { ToolError } from '@opentabs-dev/plugin-sdk';

export const getChannel = defineTool({
  // ...
  handle: async (params) => {
    const channel = findChannel(params.name);
    if (!channel) {
      throw new ToolError('Channel not found', 'CHANNEL_NOT_FOUND');
    }
    return channel;
  },
});
```

### Constructor

```typescript
new ToolError(message: string, code: string)
```

| Parameter | Type | Description |
| --- | --- | --- |
| `message` | `string` | Human-readable error message. |
| `code` | `string` | Machine-readable error code (e.g., `CHANNEL_NOT_FOUND`, `UNAUTHORIZED`, `RATE_LIMITED`). AI agents can use this to decide how to handle the error. |

Unhandled errors (non-`ToolError` exceptions) are caught by the platform, sanitized (file paths, IPs, and URLs stripped), and returned as generic error responses.

## Exports

The `@opentabs-dev/plugin-sdk` package exports:

| Export | Kind | Description |
| --- | --- | --- |
| `OpenTabsPlugin` | Class | Abstract base class for plugins |
| `defineTool()` | Function | Type-safe tool definition factory |
| `ToolError` | Class | Structured error class for tool handlers |
| `ToolDefinition` | Type | TypeScript interface for tool definitions |
| `Manifest` | Type | TypeScript type for `opentabs-plugin.json` |
| `ManifestTool` | Type | TypeScript type for a single tool in the manifest |
| `LucideIconName` | Type | String union of valid Lucide icon names |
| `LUCIDE_ICON_NAMES` | Set | Runtime set of valid Lucide icon names |
| `NAME_REGEX` | RegExp | Pattern for valid plugin names |
| `RESERVED_NAMES` | Set | Set of reserved plugin name strings |
| `validatePluginName()` | Function | Validates a plugin name, returns error string or `null` |
| `validateUrlPattern()` | Function | Validates a Chrome match pattern, returns error string or `null` |
