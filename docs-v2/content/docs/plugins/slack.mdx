## Authentication

The Slack plugin extracts session credentials automatically from the Slack web app running in your browser. It reads authentication data from Slack's local configuration, boot data, and internal state. The user just needs to be logged into Slack in Chrome â€” no API tokens, OAuth apps, or bot users required.

The plugin's `isReady()` probe verifies that valid session credentials are available before reporting the plugin as ready. If the user is not logged in or the Slack tab is on a login page, the plugin reports as `unavailable`.

## Tools

All tools are exposed to MCP clients with the `slack_` prefix (e.g., `slack_send_message`, `slack_list_channels`).

### Messages

| Tool | MCP Name | Description |
| --- | --- | --- |
| Send Message | `slack_send_message` | Send a message to a channel or thread. Supports Slack mrkdwn formatting. |
| Edit Message | `slack_edit_message` | Edit an existing message. Caller must be the original author. |
| Delete Message | `slack_delete_message` | Delete a message. Caller must be the author or channel must allow deletion. |
| Read Messages | `slack_read_messages` | Read recent messages from a channel with optional date-range filtering and pagination. |
| Read Thread | `slack_read_thread` | Read all replies in a thread including the parent message, with pagination. |
| Search Messages | `slack_search_messages` | Search messages across channels. Supports Slack search modifiers (e.g., `from:@user in:#channel`). |

### Channels

| Tool | MCP Name | Description |
| --- | --- | --- |
| List Channels | `slack_list_channels` | List workspace channels with optional type filtering and pagination. |

| Set Channel Topic | `slack_set_channel_topic` | Set a channel's topic (max 250 chars). |

## Usage Examples

### Send a message to a channel

```bash
# AI agent calls slack_send_message
{
  "channel": "C01234567",
  "text": "Hello from OpenTabs! :wave:"
}
```

### Reply in a thread

```bash
# AI agent calls slack_send_message with thread_ts
{
  "channel": "C01234567",
  "text": "Here's the follow-up you asked for.",
  "thread_ts": "1234567890.123456"
}
```

### Search across channels

```bash
# AI agent calls slack_search_messages
{
  "query": "from:@alice in:#engineering deployment",
  "sort": "timestamp",
  "sort_dir": "desc"
}
```

### Read recent messages with date filtering

```bash
# AI agent calls slack_read_messages
{
  "channel": "C01234567",
  "limit": 50,
  "oldest": "1708300800.000000"
}
```

### Upload a file

```bash
# AI agent calls slack_upload_file
{
  "channel": "C01234567",
  "content": "# Report\n\nQuarterly metrics...",
  "filename": "report.md",
  "title": "Q1 Report",
  "initial_comment": "Here's the report you requested."
}
```

## Key Concepts

### Channel and User IDs

Slack uses opaque IDs for channels (`C01234567`) and users (`U01234567`). Use `slack_list_channels` to find channel IDs by name, and `slack_list_users` or `slack_list_members` to find user IDs.

### Message Timestamps

Message timestamps (e.g., `1234567890.123456`) serve as unique message identifiers within a channel. They're returned by `slack_send_message` and `slack_read_messages`, and required by tools that operate on specific messages (edit, delete, react, pin, thread replies).

### Pagination

Tools that return lists support cursor-based pagination. When more results are available, the response includes a `response_metadata.next_cursor` field. Pass this cursor back to fetch the next page.

### Slack mrkdwn Formatting

Message text supports Slack's mrkdwn syntax: `*bold*`, `_italic_`, `~strikethrough~`, `` `code` ``, ` ```code block``` `, `>quote`, and `<@U01234567>` for user mentions.
