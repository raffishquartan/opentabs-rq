## Network Capture

Network capture uses the Chrome DevTools Protocol (`chrome.debugger`) to intercept requests and responses.

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Tool</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`browser_enable_network_capture`</Table.Cell>
      <Table.Cell>Start capturing network requests and responses for a tab. Captures URL, method, status, headers, request/response bodies (text-based), MIME type, and timing. One capture session per tab. Also enables console log capture. Also captures WebSocket frame payloads (sent and received) — retrieve them with browser_get_websocket_frames.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`browser_get_network_requests`</Table.Cell>
      <Table.Cell>Retrieve captured network requests. Useful for reverse-engineering API shapes and understanding data flows. Optionally clears the buffer after reading.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`browser_get_websocket_frames`</Table.Cell>
      <Table.Cell>Get WebSocket frames captured since browser_enable_network_capture was called on this tab. Each frame includes URL, direction (sent/received), data payload, opcode, and timestamp.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`browser_export_har`</Table.Cell>
      <Table.Cell>Export captured network traffic as a HAR 1.2 JSON file. Requires an active or completed capture session from browser_enable_network_capture.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`browser_disable_network_capture`</Table.Cell>
      <Table.Cell>Stop capturing and release the Chrome DevTools Protocol debugger session.</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

      <Table.Cell>Get console log messages from a tab. Requires `browser_enable_network_capture` to be active on the tab (the debugger captures both network requests and console output). Filter by level: log, warn, error, info, debug, or all. Optionally clears the buffer.</Table.Cell>

## Extension Diagnostics

These tools control and diagnose the OpenTabs Chrome extension itself. They use the `extension_` prefix instead of `browser_`.

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Tool</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_reload`</Table.Cell>
      <Table.Cell>Reload the Chrome extension. The extension briefly disconnects and automatically reconnects.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_get_state`</Table.Cell>
      <Table.Cell>Get the complete internal state of the extension: WebSocket connection status, all plugins with tab states, active network captures, offscreen document status.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_get_logs`</Table.Cell>
      <Table.Cell>Retrieve internal logs from the background script and offscreen document. Filter by level and source.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_get_side_panel`</Table.Cell>
      <Table.Cell>Get the side panel's React state and rendered HTML, or `{open: false}` if the panel is not open.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_check_adapter`</Table.Cell>
      <Table.Cell>Check adapter injection status for a plugin across all matching tabs. Reports adapter presence, hash validity, isReady() result, and tool names per tab.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`extension_force_reconnect`</Table.Cell>
      <Table.Cell>Force a WebSocket disconnect and immediate reconnection without reloading the full extension.</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>
