All seven packages are published to npm. `browser-extension` and `mcp-server` are additionally bundled with the CLI — `opentabs start` uses the bundled MCP server and auto-installs the bundled browser extension to `~/.opentabs/extension/`.

## npm Authentication (One-Time Setup)

Create a granular access token at [npmjs.com/settings/tokens/create](https://www.npmjs.com/settings/tokens/create):

- **Permissions**: Read and Write
- **Packages**: `@opentabs-dev/*`
- **Bypass 2FA**: Enabled

Save the token to `~/.npmrc`:

```bash
echo '//registry.npmjs.org/:_authToken=<YOUR_TOKEN>' > ~/.npmrc
```

Verify it works:

```bash
npm whoami
```

## Publishing a Release

The `scripts/publish.ts` script handles the entire publish workflow:

```bash
npx tsx scripts/publish.ts 0.0.17
```

The script:

1. Verifies npm authentication via `npm whoami`
2. Runs `npm run build` to compile all packages
3. Bumps `"version"` in all seven `platform/*/package.json` files
4. Updates cross-references between packages (e.g., `plugin-sdk` depends on `shared`)
5. Runs `npm run build` again with the new versions
6. Publishes in dependency order: `shared` → `browser-extension` → `mcp-server` → `plugin-sdk` → `plugin-tools` → `cli` → `create-plugin`

Six packages use `"publishConfig": { "access": "restricted" }` explicitly. `browser-extension` omits `publishConfig` but is still restricted — npm defaults to restricted access for scoped packages. Only `@opentabs-dev` org members can install any of them.

## After Publishing

After publishing new platform package versions, update plugin dependencies:

```bash
cd plugins/slack
# Update @opentabs-dev/* versions in package.json
npm install
npm run build
```

The `opentabs-plugin build` command embeds the installed SDK version as `sdkVersion` in `dist/tools.json`. The MCP server checks this at discovery time — if a plugin was built with a newer SDK than the server, it is rejected with a clear rebuild message.

## Version Compatibility

The MCP server uses major.minor comparison for SDK version checks:

- A plugin's major.minor must be **less than or equal to** the server's major.minor
- Patch differences are always fine
- Plugins without `sdkVersion` in `tools.json` (predating this feature) load with a warning

This means you can publish a patch release (e.g., `0.0.16` → `0.0.17`) and existing plugins continue to work without rebuilding. A minor or major bump (e.g., `0.0.x` → `0.1.0`) requires plugins to be rebuilt with the new SDK.

## What NOT to Publish

- **`plugins/*`** — example plugins are standalone projects that publish independently to npm under `opentabs-plugin-*` names
