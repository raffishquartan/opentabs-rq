Prompts let your plugin expose reusable message templates that AI agents can render with structured arguments. When an agent calls `prompts/get`, your prompt's `render()` function runs in the browser page context and returns an array of messages.

## Import

```typescript
import { definePrompt } from '@opentabs-dev/plugin-sdk';
import type { PromptDefinition, PromptArgument, PromptMessage } from '@opentabs-dev/plugin-sdk';
```

## `definePrompt(config)`

Type-safe factory function that returns a `PromptDefinition`. Identity function — provides type checking with no runtime behavior.

```typescript
// Plain usage — args are untyped Record<string, string>
function definePrompt(config: PromptDefinition): PromptDefinition;

// Typed usage — when `args` Zod schema is provided, render() receives typed args
function definePrompt<TArgs extends z.ZodObject<z.ZodRawShape>>(
  config: TypedPromptConfig<TArgs>,
): PromptDefinition;
```

When you pass an `args` Zod schema, the `render()` function receives a typed object inferred from the schema instead of a plain `Record<string, string>`. The `arguments` metadata array is auto-generated from the schema during build if not explicitly provided.

## `PromptDefinition`

```typescript
interface PromptDefinition {
  name: string;
  description?: string;
  args?: z.ZodObject<z.ZodRawShape>;
  arguments?: PromptArgument[];
  render(args: Record<string, string>): Promise<PromptMessage[]>;
}
```

## Name Prefixing

Prompt names are automatically prefixed with `<plugin>_` for MCP registration. A prompt named `compose_message` in a plugin named `slack` becomes `slack_compose_message` at the MCP level. You define just the unprefixed name.

Prompt names must match `^[a-z0-9][a-z0-9_-]*$` — start with a lowercase letter or digit; may contain lowercase letters, digits, underscores, and hyphens.

## Example

### Simple Prompt

```typescript
import { definePrompt } from '@opentabs-dev/plugin-sdk';
import type { PromptMessage } from '@opentabs-dev/plugin-sdk';

export const composeMessage = definePrompt({
  name: 'compose_message',
  description: 'Draft a Slack message for a given topic and tone.',
  arguments: [
    { name: 'topic', description: 'What the message is about', required: true },
    { name: 'tone', description: 'Tone of the message (casual, formal, urgent)' },
  ],

  async render(args: Record<string, string>): Promise<PromptMessage[]> {
    const tone = args.tone ?? 'casual';
    return [
      {
        role: 'user',
        content: {
          type: 'text',
          text: `Draft a Slack message about: ${args.topic}\nTone: ${tone}\nKeep it concise — 2-3 sentences max.`,
        },
      },
    ];
  },
});
```

### Multi-Message Prompt with Live Data

Prompts can fetch live data from the page and return multiple messages:

```typescript
import { definePrompt, fetchJSON } from '@opentabs-dev/plugin-sdk';
import type { PromptMessage } from '@opentabs-dev/plugin-sdk';

export const summarizeThread = definePrompt({
  name: 'summarize_thread',
  description: 'Summarize a Slack thread given its URL.',
  arguments: [
    { name: 'thread_url', description: 'URL of the Slack thread', required: true },
  ],

  async render(args: Record<string, string>): Promise<PromptMessage[]> {
    // Fetch thread messages from the page context
    const data = await fetchJSON<{ messages: Array<{ text: string; user: string }> }>(
      `/api/conversations.replies?thread_ts=${extractTs(args.thread_url)}`,
    );

    if (!data) {
      return [];
    }

    const threadText = data.messages
      .map((m) => `${m.user}: ${m.text}`)
      .join('\n');

    return [
      {
        role: 'user',
        content: {
          type: 'text',
          text: `Here is a Slack thread:\n\n${threadText}\n\nSummarize the key points and any action items.`,
        },
      },
    ];
  },
});
```

### Typed Prompt with Zod Schema

Use the `args` field to get type-safe arguments in your `render()` function. The `arguments` metadata array is auto-generated from the schema during build:

```typescript
import { definePrompt } from '@opentabs-dev/plugin-sdk';
import { z } from 'zod';

export const composeMessage = definePrompt({
  name: 'compose_message',
  description: 'Draft a Slack message for a given topic and tone.',
  args: z.object({
    topic: z.string().describe('What the message is about'),
    tone: z.string().describe('Tone of the message (casual, formal, urgent)').optional(),
  }),

  // args is typed as { topic: string; tone?: string } — inferred from the schema
  async render(args) {
    const tone = args.tone ?? 'casual';
    return [
      {
        role: 'user',
        content: {
          type: 'text',
          text: `Draft a Slack message about: ${args.topic}\nTone: ${tone}\nKeep it concise — 2-3 sentences max.`,
        },
      },
    ];
  },
});
```

### Registering on the Plugin Class

Assign your prompts to the `prompts` property on your plugin class:

```typescript
import { OpenTabsPlugin } from '@opentabs-dev/plugin-sdk';
import type { ToolDefinition, PromptDefinition } from '@opentabs-dev/plugin-sdk';
import { sendMessage } from './tools/send-message.js';
import { composeMessage } from './prompts/compose-message.js';

class SlackPlugin extends OpenTabsPlugin {
  readonly name = 'slack';
  // ... other properties
  readonly tools: ToolDefinition[] = [sendMessage];
  readonly prompts: PromptDefinition[] = [composeMessage];

  async isReady(): Promise<boolean> {
    // ...
  }
}

export default new SlackPlugin();
```

## Dispatch Flow

Prompt gets follow the same dispatch pipeline as tool calls:

1. AI agent calls `prompts/get` with the prefixed name and arguments
2. MCP server strips the prefix and forwards to the Chrome extension via WebSocket
3. Extension routes to the correct tab's adapter
4. Adapter calls `render(args)` in the page context
5. Messages flow back through the chain

Unlike tool dispatch, prompt gets do not support progress reporting or lifecycle hooks.
