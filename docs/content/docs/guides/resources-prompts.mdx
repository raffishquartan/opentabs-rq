Most plugins only need tools — actions the AI agent can invoke. But the MCP protocol defines two more capabilities that let your plugin expose **data** and **templates** to the agent. Resources are read-only data the agent can browse. Prompts are pre-built message templates the agent can render.

## When to Use What

Before diving in, here's the distinction:

- **Tools** are actions — "send a message", "create a page", "search for users"
- **Resources** are data — "list of channels", "current user profile", "project settings"
- **Prompts** are templates — "summarize this thread", "draft a reply", "generate a status update"

Most plugins use tools exclusively, and that's fine. Add resources when you want to give the agent browsable context without requiring an explicit tool call. Add prompts when you have reusable message patterns that benefit from structured arguments.

## Resources

A resource exposes read-only data from the web application. When an AI agent calls `resources/read`, your resource's `read()` function runs in the browser page context — with full access to the page's cookies, local storage, and JavaScript globals.

### Defining a Resource

```typescript
import { defineResource } from '@opentabs-dev/plugin-sdk';
import type { ResourceContent } from '@opentabs-dev/plugin-sdk';

const channelList = defineResource({
  uri: 'slack://channels',
  name: 'Channel List',
  description: 'Returns all Slack channels the user has access to',
  mimeType: 'application/json',

  async read(uri: string): Promise<ResourceContent> {
    const response = await fetch('/api/conversations.list', {
      credentials: 'include',
    });
    const data = await response.json();

    return {
      uri: 'slack://channels',
      text: JSON.stringify(data.channels),
      mimeType: 'application/json',
    };
  },
});
```

### ResourceContent Return Type

The `read()` function returns a `ResourceContent` object:

```typescript
interface ResourceContent {
  uri: string;       // Must match the resource URI
  text?: string;     // Text content (for JSON, plain text, HTML, etc.)
  blob?: string;     // Base64-encoded binary content (for images, PDFs, etc.)
  mimeType?: string; // Content type of this response
}
```

Return either `text` or `blob`, not both. Most resources return `text` with JSON data — this gives the AI agent structured data it can reason about.

### Typed Resources with Schema

Resources can include an optional Zod `schema` to document the shape of their content. The schema is a type-level contract — the runtime dispatch path does not validate against it:

```typescript
import { defineResource, fetchJSON } from '@opentabs-dev/plugin-sdk';
import type { ResourceContent } from '@opentabs-dev/plugin-sdk';
import { z } from 'zod';

const channelList = defineResource({
  uri: 'myapp://channels',
  name: 'Channel List',
  description: 'All active channels',
  mimeType: 'application/json',
  schema: z.object({
    channels: z.array(z.object({ id: z.string(), name: z.string() })),
  }),

  async read(uri: string): Promise<ResourceContent> {
    const data = await fetchJSON<{ channels: { id: string; name: string }[] }>('/api/channels');
    return { uri: 'myapp://channels', text: JSON.stringify(data), mimeType: 'application/json' };
  },
});
```

When a `schema` is provided, use `ResourceContentType<typeof channelList>` to derive the TypeScript content type for use elsewhere in your plugin. See [SDK Reference: Resources](/docs/sdk/resources) for the full API.

### Registering Resources on Your Plugin

Add your resources to the `resources` array on your plugin class:

```typescript
import { OpenTabsPlugin } from '@opentabs-dev/plugin-sdk';
import type { ResourceDefinition } from '@opentabs-dev/plugin-sdk';

class MyPlugin extends OpenTabsPlugin {
  override readonly name = 'my-plugin';
  override readonly resources: ResourceDefinition[] = [channelList, userProfile];
  // ... tools, isReady, etc.
}
```

`override` is required because TypeScript's `noImplicitOverride` rule (enabled in the project's tsconfig) requires the keyword when overriding any inherited class member. Omitting `override` causes a TS4114 type error at build time.

### How Resources Are Exposed to AI Agents

The platform prefixes resource URIs to prevent collisions between plugins. Your resource URI `slack://channels` becomes:

```
opentabs+slack://slack://channels
```

The AI agent sees the prefixed URI when listing resources. When it reads the resource, the platform strips the prefix and dispatches to your `read()` function with the original URI.

## Prompts

A prompt defines a reusable message template with structured arguments. When an AI agent calls `prompts/get`, your prompt's `render()` function runs in the browser page context and returns formatted messages.

Prompts are useful when you have a pattern the agent uses repeatedly — like summarizing a thread, drafting a reply in a specific format, or generating a report from page data.

### Defining a Prompt

```typescript
import { definePrompt } from '@opentabs-dev/plugin-sdk';
import type { PromptMessage } from '@opentabs-dev/plugin-sdk';

const summarizeThread = definePrompt({
  name: 'summarize_thread',
  description: 'Generates a prompt to summarize a Slack thread with key decisions and action items',
  arguments: [
    { name: 'channel', description: 'Channel name or ID', required: true },
    { name: 'thread_ts', description: 'Thread timestamp', required: true },
    { name: 'style', description: 'Summary style: brief, detailed, or action-items', required: false },
  ],

  async render(args: Record<string, string>): Promise<PromptMessage[]> {
    const channel = args['channel'];
    const threadTs = args['thread_ts'];
    const style = args['style'] ?? 'brief';

    // Fetch the thread data from the page context
    const response = await fetch(
      `/api/conversations.replies?channel=${channel}&ts=${threadTs}`,
      { credentials: 'include' }
    );
    const data = await response.json();
    const messages = data.messages
      .map((m: { user: string; text: string }) => `${m.user}: ${m.text}`)
      .join('\n');

    return [
      {
        role: 'user',
        content: {
          type: 'text',
          text: `Summarize this Slack thread (style: ${style}):\n\n${messages}`,
        },
      },
    ];
  },
});
```

### PromptMessage Return Type

The `render()` function returns an array of `PromptMessage` objects:

```typescript
interface PromptMessage {
  role: 'user' | 'assistant';
  content: { type: 'text'; text: string };
}
```

Return one or more messages that set up the conversation. A common pattern is a single `user` message with the full prompt text. For more complex templates, return multiple messages to establish context:

```typescript
async render(args) {
  return [
    {
      role: 'user',
      content: { type: 'text', text: 'You are a technical writer for our team.' },
    },
    {
      role: 'assistant',
      content: { type: 'text', text: 'I\'ll help you write clear, concise documentation.' },
    },
    {
      role: 'user',
      content: { type: 'text', text: `Write release notes for version ${args['version']}.` },
    },
  ];
}
```

### Typed Prompts with Zod Schema

Instead of manually defining the `arguments` array, you can pass a Zod `args` schema. The MCP metadata (argument names, descriptions, required flags) is auto-generated from the schema at build time, and `render()` receives a typed object instead of `Record<string, string>`:

```typescript
import { definePrompt, fetchJSON } from '@opentabs-dev/plugin-sdk';
import { z } from 'zod';

const statusReport = definePrompt({
  name: 'status_report',
  description: 'Generate a project status report',
  args: z.object({
    project: z.string().describe('Project name'),
    format: z.enum(['brief', 'detailed']).describe('Report format'),
  }),
  async render(args) {
    // args is typed as { project: string; format: 'brief' | 'detailed' }
    const data = await fetchJSON<{ items: { title: string; status: string }[] }>('/api/items');
    const summary = data.items.map(item => `- [${item.status}] ${item.title}`).join('\n');

    return [
      {
        role: 'user',
        content: {
          type: 'text',
          text: `Status for ${args.project} (${args.format}):\n\n${summary}`,
        },
      },
    ];
  },
});
```

Use `.describe()` on each schema field to provide descriptions — these become the argument descriptions in the MCP metadata. Fields without `.describe()` are registered with no description. All fields are marked as required unless wrapped with `.optional()`.

### Registering Prompts on Your Plugin

Add your prompts to the `prompts` array on your plugin class:

```typescript
class MyPlugin extends OpenTabsPlugin {
  override readonly name = 'my-plugin';
  override readonly prompts: PromptDefinition[] = [summarizeThread, draftReply];
  // ... tools, resources, isReady, etc.
}
```

`override` is required because TypeScript's `noImplicitOverride` rule (enabled in the project's tsconfig) requires the keyword when overriding any inherited class member. Omitting `override` causes a TS4114 type error at build time.

### How Prompts Are Exposed to AI Agents

Prompt names are prefixed with your plugin name to prevent collisions. Your prompt `summarize_thread` becomes:

```
my-plugin_summarize_thread
```

The AI agent sees the prefixed name when listing prompts and uses it to call `prompts/get`.

## The Dispatch Flow

Resources and prompts follow the same dispatch pipeline as tools:

<DispatchFlow />

1. The AI agent requests `resources/read` or `prompts/get`
2. The MCP server looks up the resource/prompt in the registry, finds the owning plugin
3. The server dispatches to the Chrome extension via WebSocket
4. The extension finds a matching tab for the plugin and executes the `read()` or `render()` function in the page's MAIN world
5. The result flows back through the same chain to the AI agent

Two differences from tool dispatch:

- **No progress reporting** — resources and prompts don't support the `reportProgress` callback. If a resource read or prompt render takes a long time, it runs until it completes or hits the dispatch timeout.
- **No lifecycle hooks** — `onToolInvocationStart` and `onToolInvocationEnd` only fire for tools, not for resource reads or prompt renders.

## Full Plugin Example

Here's a complete plugin that uses all three capabilities — tools, resources, and prompts:

```typescript
import {
  OpenTabsPlugin,
  defineTool,
  defineResource,
  definePrompt,
  fetchJSON,
  ToolError,
  log,
} from '@opentabs-dev/plugin-sdk';
import type {
  ToolDefinition,
  ResourceDefinition,
  PromptDefinition,
} from '@opentabs-dev/plugin-sdk';
import { z } from 'zod';

// A resource that lists project items
const itemList = defineResource({
  uri: 'tracker://items',
  name: 'Project Items',
  description: 'Returns all items in the current project',
  mimeType: 'application/json',
  async read(uri: string) {
    const data = await fetchJSON<{ items: { id: string; title: string; status: string }[] }>(
      '/api/items'
    );
    return {
      uri: 'tracker://items',
      text: JSON.stringify(data.items),
      mimeType: 'application/json',
    };
  },
});

// A prompt that generates a status report
const statusReport = definePrompt({
  name: 'status_report',
  description: 'Generates a prompt to create a project status report',
  arguments: [
    { name: 'period', description: 'Report period (e.g., "this week")', required: true },
  ],
  async render(args) {
    const data = await fetchJSON<{ items: { title: string; status: string }[] }>(
      '/api/items'
    );
    const summary = data.items
      .map(item => `- [${item.status}] ${item.title}`)
      .join('\n');

    return [
      {
        role: 'user' as const,
        content: {
          type: 'text' as const,
          text: `Generate a status report for ${args['period']}. Here are the current items:\n\n${summary}`,
        },
      },
    ];
  },
});

// A tool that creates a new item
const createItem = defineTool({
  name: 'create_item',
  description: 'Creates a new item in the project tracker',
  input: z.object({
    title: z.string().describe('Item title'),
    assignee: z.string().optional().describe('Who to assign the item to'),
  }),
  output: z.object({
    id: z.string(),
    title: z.string(),
    url: z.string(),
  }),
  async handle(params) {
    const item = await fetchJSON<{ id: string; title: string; url: string }>(
      '/api/items',
      { method: 'POST', body: JSON.stringify(params) }
    );
    log.info('Created item', { id: item.id, title: item.title });
    return item;
  },
});

export default class TrackerPlugin extends OpenTabsPlugin {
  override readonly name = 'tracker';
  override readonly version = '1.0.0';
  override readonly displayName = 'Project Tracker';
  override readonly description = 'Manage project items';
  override readonly urlPatterns = ['*://tracker.example.com/*'];
  override readonly tools: ToolDefinition[] = [createItem];
  override readonly resources: ResourceDefinition[] = [itemList];
  override readonly prompts: PromptDefinition[] = [statusReport];

  override async isReady(): Promise<boolean> {
    return document.querySelector('.app-loaded') !== null;
  }
}
```

The AI agent can now:

1. **Browse context** — read `tracker://items` to see what items exist
2. **Take action** — call `create_item` to add a new item
3. **Generate reports** — use the `status_report` prompt to create a formatted summary

## Best Practices

**Resources:**

- Return JSON for structured data — it's the most useful format for AI agents
- Include only the data the agent needs. A large, unfiltered response wastes context.
- Set `mimeType` so the agent knows how to interpret the content
- Use SDK utilities like `fetchJSON` for fetching data instead of raw `fetch`

**Prompts:**

- Fetch live data in `render()` — prompts are most powerful when they include current page state, not just static templates
- Use arguments for variation (style, format, scope) rather than creating separate prompts for each variation
- Return the minimum messages needed — usually one `user` message is sufficient

**General:**

- Use tools for actions, resources for data, prompts for templates. If you're unsure, use a tool — it's the most flexible.
- Resources and prompts are optional. Many great plugins only define tools.
